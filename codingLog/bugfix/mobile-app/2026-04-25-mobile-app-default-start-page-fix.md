# Mobile App Default Start Page Rollback Note

## 基本信息

- 日期：2026-04-25
- 分类：bugfix
- 模块：mobile-app
- 相关文件：`mobile-app/src/app/App.tsx`

## 当前现象

- 页面初始化后，首屏仍可能停留在 AI 对话页。
- 此时向右滑会出现页面消失，随后界面失去可操作性的更严重异常。

## 本次处理

- 已回退本轮对横向分页容器的结构性修改。
- 已恢复到原来的“双屏横向容器 + `viewportResetKey` + `resetToDashboard()`”实现。
- 已重新执行 `npx vite build`，让 `dist` 与回退后的源码保持一致。

## 回退原因

- 本轮尝试将分页容器改为“单视口 + 绝对定位面板”，目的是消除视觉页与状态页不一致的问题。
- 实际结果更差：初始化仍可能落在对话页，且首次向右滑后会把页面拖出可视区域，导致界面无法继续操作。
- 说明这次修改没有命中真正根因，反而破坏了现有拖拽边界与位移模型，因此已撤回。

## 关键结论

- 问题目前仍未解决。
- 现阶段不能再把问题简单归因为 `200vw` 布局本身，真实根因更可能在以下几类状态的叠加：
  - `motion` 拖拽状态与 React `currentView` 状态不同步。
  - 初始化阶段的 `transform` 恢复时序与 `key` 重挂载时序存在竞争。
  - 浏览器或 WebView 的原生滚动恢复、回前台恢复与动画状态恢复互相影响。

## 后续排查重点

- 记录初始化时分页容器的真实 `transform`、`currentView` 和 `window.scrollX`，先确认到底是哪一层先跑偏。
- 单独验证 `viewportResetKey` 重挂载是否会让 Motion 在首次手势后重新建立拖拽基线。
- 区分“浏览器把文档滚到右侧”和“Motion 把容器平移到左侧”这两类问题，不再混在一起判断。
