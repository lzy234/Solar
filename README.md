# version 1 finished
项目背景：客户在上海多地投资了太阳能光伏电站项目，但是电站与电站的信息运维平台不一样，对客户来说不方便查看电站信息和告警。
现阶段有三个需求：
1、把光伏电站的运维平台整合起来，客户说可以要到API，所以我们只需要后端适配不同的API，统一在一个前端展示就行了；
2、可能要搞一些自动化的流程，帮客户接入一个OpenClaw，作为客户的秘书，可能会写写电站运营周报，推送告警信息到客户的通讯软件等功能；
3、光伏电站的异常检测，异常情况比如：有脏东西遮盖住了太阳能板，导致发电量下降收益下降等。目前阶段还未制定具体的检测方案，但是我们大致希望通过数据分析和简单硬件加装来完成这一点。
现阶段目标：客户经理与客户做过简单的沟通，但是没有拿出看起来靠谱点的文档、PPT之类的大致方案，现在就需要你来做这第一步工作。

# Demo 部署（GitHub Pages）finished
已配置自动发布工作流：`.github/workflows/pages.yml`。

你只需要：
1、把仓库 push 到 GitHub（`main` 或 `master` 分支）；
2、在 GitHub 仓库设置中进入 `Settings -> Pages`，将 Source 设为 `GitHub Actions`（只需一次）；
3、后续每次 push 会自动发布 `demo/` 目录。

访问地址通常为：`https://<你的GitHub用户名>.github.io/<仓库名>/`

# version 3 finished
现在经过与客户的初步沟通，确定了以下场景与需求：
1、用户所建的光伏电站有不同的运维后台，而用户可以开放这些后台的API给我们；
2、用户不想要数据看板，我设想最好的解决方案是，把数据直接推送到飞书；
3、光伏发电是两组光伏板并联，可以得到的数据是逆变器检测的两组板的电压与电流实时数据，通常来说，判断发电板是否异常可以通过比对电流差异是否长期相差较大来判断，但实现这一点可能需要一些数据分析的算法设计；
4、agent方面：
    a.需要为客户定制一个OpenClaw，结合数据分析算法来为客户进行告警提醒；
    b.需要做一些日常运营的任务，生成一些周报月报之类的；
    c.需要一个工单系统，在新告警生成时检索过去的维护经验，作出告警的处理建议。
