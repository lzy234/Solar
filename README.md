# version 1 finished
项目背景：客户在上海多地投资了太阳能光伏电站项目，但是电站与电站的信息运维平台不一样，对客户来说不方便查看电站信息和告警。

# 需求分析
现在经过与客户的初步沟通，确定了以下场景与需求：
1、用户所建的光伏电站有不同的运维后台，而用户可以开放这些后台的API给我们；
2、用户不想要数据看板，为了方便给用户报价，最好还是设计一个前端页面，前端页面的设计要求如下：
    a.不采用传统的电站运维后台布局，主要展示一些运维的宏观数据；
    b.增加一个LLM对话框，用于回答用户一些运维相关的问题；
    c.需要一个关于告警的推送界面，该界面展示告警信息，以及处理建议，其后端由Open Claw提供支撑。
3、光伏发电是两组光伏板并联，可以得到的数据是逆变器检测的两组板的电压与电流实时数据，通常来说，判断发电板是否异常可以通过比对电流差异是否长期相差较大来判断，但实现这一点可能需要一些数据分析的算法设计；
4、agent方面：
    a.需要为客户定制一个OpenClaw，结合数据分析算法来为客户进行告警提醒；
    b.需要做一些日常运营的任务，生成一些周报月报之类的；
    c.需要一个工单系统，在新告警生成时检索过去的维护经验，作出告警的处理建议。

# 任务分配
1、前端+Agent后端 分给一个人，以下是补充说明：
    a.前端页面的设计需要采用简单的方式，例如把飞书的多维表格直接转换成H5页面，可以参考这个页面https://ebooktst.consultech.com.cn/pah/；
    b.Open Claw需要能在以下情况主动发送消息给客户：有告警时，客户询问时；
    c.告警处理流程：数据分析算法识别异常 -> 触发后端 Agent -> Agent 检索工单库 -> 综合推理生成处理建议 -> 推送告警与处理建议至客户端
    d.日常运维处理流程：用户提问->agent检索相关数据->根据用户需求给出回答。
2、数据分析与工单系统 分给另一个人，以下是补充说明：
    a.数据分析需要先处理API拿到的数据，数据量不大，前期实现要考虑最快成本最低的方式；
    b.异常告警算法的设计并不复杂，通过电流比对，比对时间段内的方差即可，最后要输出异常的相关信息；
    c.工单系统初期暂时不实现，但是思路要点如下：
        (1)要设计一套格式化的工单供维修人员填写
        (2)要将工单反馈存储起来，并做向量化，供Agent检索
