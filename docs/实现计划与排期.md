# 实现计划与排期

## 1. 实现计划

### 1.1 数据采集层

**目标**：打通各电站 API，统一数据格式供上层使用。

- 开发多源 API 适配器，对接不同厂商光伏电站运维后台
- 实现定时拉取与标准化处理流程
- 输出统一数据结构（设备 ID、时间戳、电流/电压/发电量等字段）

### 1.2 异常检测模块

**目标**：通过方差分析自动识别发电板异常。

- 针对并联光伏板组，计算特定时间窗口内电流的方差差异
- 设定告警阈值，超过时输出：异常设备 ID、异常时间、数值特征
- 触发下游告警推送流程

### 1.3 Agent 后端

**目标**：实现智能推送、问答与报表自动化。

- **异常告警流**：接收异常信号 → 检索历史工单 → 调用 LLM 生成处理建议 → 推送至前端及飞书/微信
- **用户问答流**：接收用户提问 → 结合电站实时数据与知识库 → 生成回答
- **自动报表**：定时触发，调用 Agent 生成结构化周报/月报并推送管理人员

### 1.4 前端

**目标**：提供简洁易用的运维交互界面（简约 H5 风格）。

- **宏观数据看板**：实时展示电站运行状态、发电达成率等核心指标
- **LLM 对话组件**：嵌入式对话框，对接 Agent 问答接口
- **告警流展示**：卡片式流式展示告警内容及处理建议

### 1.5 工单与知识库（分阶段）

**目标**：沉淀历史维护经验，提升 Agent 决策质量。

- 第一阶段：设计结构化维修反馈表格，建立格式化存储流程
- 第二阶段：对历史工单进行文本清洗与向量化（Embedding），存入向量数据库
- 第三阶段：为 Agent 提供 RAG 检索接口，通过语义匹配辅助告警决策

---

## 2. 项目排期

项目分三期推进，客户可按阶段验收交付，便于阶段性报价与预期管理。

---

### 第一期：单站全流程贯通（约 3 周）

**目标**：接入一个电站的真实 API，跑通从数据采集到前端展示、告警推送的完整链路，作为基础版本交付。

| 周次 | 时间 | 任务 |
|------|------|------|
| Week 1 | 04/13 ~ 04/19 | 单站 API 接入；数据标准化处理；基础异常检测算法 |
| Week 2 | 04/20 ~ 04/26 | Agent 告警流搭建（LLM 调用 + 飞书/微信推送）；Agent 用户问答流 |
| Week 3 | 04/27 ~ 05/03 | 前端看板、对话框、告警流展示开发；全流程联调与交付 |

**交付物**：可运行的单站运维平台，包含数据看板、智能告警推送、LLM 问答功能。

---

### 第二期：告警算法迭代与 Agent 工作流优化（约 3 周）

**目标**：在真实数据反馈基础上打磨算法精度，完善 Agent 工作流的稳定性与智能程度，支持自动报表。

| 周次 | 时间 | 任务 |
|------|------|------|
| Week 4 | 05/04 ~ 05/10 | 告警阈值调优；异常检测算法迭代（基于第一期真实数据） |
| Week 5 | 05/11 ~ 05/17 | Agent 工作流优化（提示词、上下文、推送策略）；自动周报/月报功能 |
| Week 6 | 05/18 ~ 05/24 | 多站 API 适配扩展；压测与稳定性优化；阶段交付 |

**交付物**：告警准确率提升、Agent 工作流稳定版本，支持多电站接入与自动化报表。

---

### 第三期：工单系统接入 Agent 工作流（约 2 周）

**目标**：将历史维修工单沉淀为知识库，接入 Agent，使告警处理建议具备经验参考能力。

| 周次 | 时间 | 任务 |
|------|------|------|
| Week 7 | 05/25 ~ 05/31 | 工单表格设计与格式化存储；历史数据清洗与向量化（Embedding） |
| Week 8 | 06/01 ~ 06/07 | RAG 检索接口开发；Agent 告警流集成工单知识库；系统验收交付 |

**交付物**：具备 RAG 能力的完整 Agent 工作流，告警建议可参考历史工单经验。
