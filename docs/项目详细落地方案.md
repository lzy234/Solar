# 项目详细落地方案

## 1. 需求分析

### 1.1 项目背景与目标
针对客户在多地投资的太阳能光伏电站运维平台不统一的问题，建设一个集成的运维管理平台。通过接入各电站 API，实现数据的宏观展示、智能告警推送及 LLM 交互问答。

### 1.2 核心需求
1.  **多平台接入**：通过 API 接入不同厂商的光伏电站运维后台数据。
2.  **非传统前端交互**：
    *   展示运维宏观数据，避开传统复杂布局。
    *   集成 LLM 对话框，提供实时运维咨询。
    *   实时告警推送界面，包含异常信息及由 OpenClaw 支撑的处理建议。
3.  **智能异常检测**：利用数据分析算法，通过对比并联光伏板的实时电流差异（方差分析）来判断发电板是否异常。
4.  **智能 Agent**：
    *   主动推送：在发生告警或客户提问时主动发送消息。
    *   日常任务：自动生成运维周报与月报。
    *   工单辅助：结合历史维护经验，为新告警提供针对性处理建议。

---

## 2. 前端+Agent后端实现计划

### 2.1 前端开发
*   **交互设计**：采用简约 H5 页面风格（参考飞书多维表格转 H5 模式），降低开发成本。
*   **功能模块**：
    *   **宏观数据看板**：实时显示电站运行状态、发电達成率等核心指标。
    *   **LLM 对话组件**：嵌入式对话框，对接 OpenClaw 接口。
    *   **告警流展示**：卡片式流式展示告警及处理建议。

### 2.2 Agent 后端开发
*   **智能推送逻辑**：
    *   **异常告警流**：接收数据分析模块的异常信号 -> 检索工单系统历史经验 -> 调用 LLM 综合生成建议 -> 推送至前端及飞书/微信。
    *   **用户问答流**：接收用户提问 -> 根据电站实时数据与知识库给出回答。
*   **自动化报表**：定时触发数据汇总，调用 Agent 生成结构化的周报、月报并推送给管理人员。

---

## 3. 数据分析与工单系统实现计划

### 3.1 数据处理与分析算法
*   **数据采集层**：开发多源 API 适配器，低成本实现各电站数据的定时拉取与标准化处理。
*   **异常检测算法**：
    *   **逻辑实现**：针对并联两组光伏板，计算特定时间段内电流的方差差异。
    *   **输出定义**：当差异超过阈值时，输出异常设备 ID、异常时间及数值特征，触发告警流程。

### 3.2 工单系统实现（分阶段）
*   **工单标准化设计**：设计结构化的维修反馈表格，供一线人员记录故障原因、处理步骤及结果。
*   **知识库构建**：
    *   **存储与向量化**：将历史工单反馈进行文本清洗，利用向量化技术（Embedding）存储至向量数据库。
    *   **检索增强 (RAG)**：为 Agent 提供检索接口，通过语义匹配寻找相似的历史故障案例，辅助决策。
*   **快速落地策略**：初期优先实现数据标准化与基础告警算法，工单系统先建立格式化存储流程，逐步完善向量化检索功能。
